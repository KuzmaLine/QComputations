    \section{Структура библиотеки}
        Идея данной библиотеки заключается в том, чтобы полностью избавить программиста от рутины реализации, оптимизации и параллелизации 
        при моделировании квантовых процессов. Сложность заключается в том, что каждый эксперимент и модель, по своей сути, уникальны. 
        Разные структуры состояний систем, разные операторы и численные методы моделирования. Данная библиотека унифицирует всё это, 
        насколько это возможно к определённым базовым понятиям, с которыми потом и работает.

        Всё начинается с понятия состояния, по своей сути, оно определяет всю систему.
        Базовым понятием состояния является класс \textbf{Basis_State}.
        Любой класс, являющийся наследником \textbf{Basis_State}, принимается библиотекой.
        Далее, я буду использовать класс \textbf{Some_State}, для обозначения подобного класса.

        Сам класс \textbf{Basis_State} включает в себя 2 главных понятия:
        \begin{enumitem}
            \item{Кудита - многозначного кубита, максимальное значение которого 
        также хранится в классе в массиве \textbf{max_vals}.}
            \item{Группа кудитов: во многих системах очень удобно разделение кудитов на группы, как с точки зрения индексации, так и 
            с точки зрения математических нотаций. (В случае, когда оператор применяется к какой-то подсистеме, к остальным кудитам 
            применяется единичный оператор)}
        \end{enumitem}

        Начальное же состояние, которое может быть суперпозицией нескольких состояний, создаётся с помощью шаблонного класса 
        \textbf{State<Some_State>}. Примеры можно посмотреть в директории библиотеки examples. На основе данного состояния будет дальше 
        генерироваться сам базис (данный процесс называется \textsf{Отбор рабочей области}).
        \textbf{Внимание! На данный момент нет автоматической нормализации данного вектора. Проверки на дураков также нет.}

        Следующее понятие, являющейся основой данной библиотеки: \textbf{Operator<Some_State>}. Данный класс предназначен для создания 
        собственных операторов и для их унификации. Работает это следующим образом.
        Вы пишите операторы в функциях следующего прототипа:
        \begin{mdframed}[linecolor=black, topline=true, bottomline=true,
            leftline=false, rightline=false]
            \begin{minted}[mathescape]{c++}
                State<Some_State> some_op(const Some_State& state);
            \end{minted}
        \end{mdframed}

        После этого, вы создаёте сам ваш оператор следующим образом:
        \begin{mdframed}[linecolor=black, topline=true, bottomline=true,
            leftline=false, rightline=false]
            \begin{minted}[mathescape]{c++}
                using OpType = Operator<Some_State>;
                OpType H = OpType(some_func_1_1) * OpType(some_func_1_2) * ... * OpType(some_func_1_n) + 
                           OpType(some_func_2_1) * OpType(some_func_2_2) * ... * OpType(some_func_2_n) +
                           ...
                           OpType(some_func_m_1) * OpType(some_func_m_2) * ... * OpType(some_func_m_n);
            \end{minted}
        \end{mdframed}

        Данный оператор передаётся в конструктор гамильтониана. Можно также использовать напрямую для прогонки конкретного состояния
         через метод \textbf{apply}.
        Примеры вышеописанного также есть в директории examples.

        Далее идёт само понятие гамильтониана. Класс-конструктор для гамильтониана от произвольного состояния: 
        \textbf{H_by_Operator<Some_State>} для одноядерной версии и \textbf{BLOCKED_H_by_Operator<Some_State>} для многоядерной. 
        В нём же генерируется базис.
        В библиотеке также есть и готовые гамильтонианы и понятия состояний, например \textbf{CHE_State} и \textbf{H_TCH}, которые описаны
        в главе \textbf{Описание готовых состояний, гамильтонианов и операторов}.
        Все эти классы являются наследниками класса \textbf{Hamiltonian} (для многоядерной версии \textbf{BLOCKED_Hamiltonian}).
        Классы родители всегда работают только с состоянием \textbf{Basis_State}, то есть, если вы вытащите базис из гамильтониана 
        методом \textbf{get_basis}, вы получите \textbf{std::set<Basis_State>}.
        
        Дальнейшее моделирование инвариантно относительно всех предыдущих понятий с точки зрения реализации и классов. Поэтому, моделирование  
        происходит с помощью отдельных функций, работающие лишь с \textbf{State<Basis_State>}, который сначала генерируется из базиса 
        гамильтониана с помощью функции \textbf{fit_to_basis}. А также с \textbf{Hamiltonian}, здесь кастинг происходит автоматически.

        На данный момент реализованы методы:
            \begin{itemize}
                \item{Моделирование уравнения Шрёдингера через спектральное разложение. (\textbf{schrodinger})}
                \item{Моделирование основного квантового уравнения с помощью метода Рунге-Кутта 4 порядка. 
                (\textbf{quantum_master_equation})}
            \end{itemize}
        Результаты этих методов в дальнейшем визуализируется либо напрямую, либо через файловую систему и дополнительный скрипт.
        Об этом подробно написано в главе \textbf{Визуализация}.

        Для более детального понимания, приведена схема зависимостей внутри стуктуры данной библиотеки. <Вставить схему>

        Библиотека, на данный момент, делится на 4 версии:
        \begin{enumitem}
            \item{Версия на одном ядре с поддержкой графиков (libQComputations_SINGLE.so)}
            \item{Версия на одном ядре без поддержки графиков (libQComputations_SINGLE_NO_PLOTS.so)}
            \item{Версия на многих ядрах с поддержкой графиков (libQComputations_CPU_CLUSTER.so)}
            \item{Версия на многих ядрах без поддержки графиков (libQComputations_CPU_CLUSTER_NO_PLOTS.so)}
        \end{enumitem}

        2 и 4 версии предполагают использование файловой системы для хранения результатов, они сохраняют в указанные программистом директории 
        файлы в формате CSV. Визуализация этих данных происходит с помощью скрипта на языке Python, путь к которому, 
        после установки библиотеки, будет хранится в переменной окружения \$SEABORN_PLOT. Инструкция использования данного скрипта
        находится в главе \textbf{Визуализация}.

        Библиотека делится на 2 типа (на данный момент): последовательная/одноядерная (1, 2) и MPI (3, 4). Их структуры абсолютно 
        аналогичны друг другу, хотя в MPI также входит одноядерная версия.
