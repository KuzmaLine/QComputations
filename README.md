# ПО для квантовых вычислений на C++

В данном репозитории лежит программная реализация на C++ для моделирования квантовых экспериментов.
Все реализации лежат в папке src.

В силу низкоуровневости MPI, некоторые решения несовсем стабильные и тяжелопереносимые, что в будущем будет исправляться. Например, переписыванием части программ на нити, использованием графических ядер или заменой на более надёжные графические интерпретаторы, чем matplotlib Python-а.

Есть 3 версии библиотеки:
1. Для одного ядра.
2. Для нескольких MPI ядер.
3. Для нескольких узлов.

В разработке версии с использованием cuda на 1 узле (Замена 2 и 3 версий)

# Действия перед компиляцией
В силу сложности, и плохой переносимости Intel-овских библиотек, принято решение ручного прописывания путей к заголовочным файлам и к динамическим библиотекам через переменные окружения. (Рекомендация - поставить Intel OneApi BaseToolkit)

Команды для установки пакетов (для каждого далее sudo ./исполняемый_файл):
1. Intel OneApi BaseToolkit - wget https://registrationcenter-download.intel.com/akdlm/IRC_NAS/992857b9-624c-45de-9701-f6445d845359/l_BaseKit_p_2023.2.0.49397_offline.sh
2. Intel OneApi DPCPP++ - wget https://registrationcenter-download.intel.com/akdlm/IRC_NAS/ebf5d9aa-17a7-46a4-b5df-ace004227c0e/l_dpcpp-cpp-compiler_p_2023.2.1.8_offline.sh
3. Intel Python (Если требуются графики) - wget https://registrationcenter-download.intel.com/akdlm/IRC_NAS/03aae3a8-623a-47cf-9655-5dd8fcf86430/l_pythoni39_oneapi_p_2023.2.0.49422_offline.sh

Перед компиляцией библиотеки необходимо установить пару переменных окружений.

Для версии на 1 ядре (Для 1 версии):
1. MKL_INCLUDE_PATH - директория с заголовочными файлами Intel MKL. (Установка - sudo apt install intel-mkl (Работает только для 1 и 2 версий, поэтому рекомендация установить OneApi))
2. PYTHON_H_INCLUDE_PATH (Если требуются графики) - путь к директории, где лежит Python.h от вашего текущего интерпретатора Python 3.9
3. PYTHON_LIB_PATH (Если требуются графики) - путь к директории с динамическими библиотеками от этого Python-а. В первую очередь libpython(что-то там).so

Для версии с MPI (Для 2 и 3 версии):
1. Всё тоже самое, что и для 1 ядра.
2. MPI_INCLUDE_PATH - директория с заголовочным файлом mpi.h (для 3 версии обязательно Intel MPI + Intel Python)

# Компиляция библиотек

Заголовочные файлы лежат в папке src. Для 1, 2 и 3 версии использовать заголовочные файлы QComputations.hpp, QComputations_MPI.hpp и QComputations_MPI_CLUSTER.hpp соответственно.

install.sh запускает 3 скрипта:
1. Для одного ядра (install_single.sh) - libQComputations_single.so (Требуется Intel MKL)
2. Для нескольких MPI ядер (install_mpi.sh) - libQComputations_mpi.so (Требуется Intel MKL, OpenMPI или IntelMPI)
3. Для нескольких узлов (install_mpi_cluster.sh) - libQComputations_mpi_cluster.so (Требуется полный пакет Intel OneApi BaseToolkit (бесплатный) + Intel DPCPP++ + Intel MPI + Intel Python)

Все эти библиотеки, с добавкой sudo, переносятся в директорию, указанную в качестве аргумента скрипту install.sh. (То есть, если написать ./install.sh "/usr/lib", то все библиотеки перенесутся в /usr/lib)

Разница между 2 и 3 библиотеками в том, что во 2 не реализована эффективное параллельное перемножение матриц. (Та же скорость, что и у обычного blas)
Собственно, если программа работает не на кластере, то 3 реализация будет не очень эффективна, в силу маленького количества ядер и бессмысленности разделения матрицы на блоки.

Компиляция программ с библиотекой также нетривиальна из-за Python API. (Проблема будет решена сильно позже)

Примеры их компиляций будут описаны в makefile в папке examples. (Не готово!)

# Описание структуры программы
(Всё лишнее из ПО будет вскоре будет удалено!)

Всё ПО находится в namespace QComputations!!!

Все hpp файлы оснащены комментариями с описанием, что делает каждая функция. (Кроме тех, в котором описание кроется в самом названии оной)
Примеры программ лежат в директории examples. - (Не готово!)

Файл config.hpp - для настройки поведения программы по умолчанию (Неудачное решение, которое скоро заменю)

Файлы matrix - реализация класса матриц. Является основой всех остальных файлов. (В разработке CSR_Matrix)

Файлы big_uint - реализация больших чисел. Используется для реализации файлов state.

Файлы cavity_state - состояние внутри одной полости. (То есть есть некоторое пространство, внутри которого находятся фотоны и атомы)

Файлы state - состояние решётки полостей. Подробности смотреть в файле state.hpp

Файлы hamiltonian - Сами гамильтонианы. Есть 6 реализаций:

1. H_JC - Гамильтониан Джеймса-Каммингса (1 полость, n фотонов, 1 атом)
2. H_TC - Гамильтониан Тависа-Каммингса (1 полость, n фотонов, m атомов)
3. H_TCH - Гамильтониан Тависа-Каммингса-Хабарда (k полостей, n фотонов, m атомов)
4. H_by_matrix - Гамильтониан, заданный с помощью комплексной матрицы.
5. H_by_func - Гамильтониан, заданного размера, реализованный с помощью функций.
6. H_Blocked - Гамильтониан, распределённый на блоки по разным процессам. (Только в 3 библиотеке) (Не готово!)

Все гамильтонианы, кроме 4, поддерживают параллельную генерацию. Для 5 и 6 - отдельная 
инструкция в examples H_by_func_MPI_example.cpp и H_Blocked_MPI_example.cpp соответственно. (Не готово!)

Файлы dynamic - функции процесса эволюции. Реализованы: 
1. Шрёдингер для статичного гамильтониана.
2. Эволюция с помощью основного квантового уравнения

Для обоих есть параллельные реализации. (ВАЖНО! - для параллельного основого квантово уравнения нужна 3 версия библиотеки, и нужно прочесть предупреждение в dynamic.hpp)

Файлы functions - дополнительные полезные функции. Подробнее в functions.hpp

Файлы graph - поиск дерева состояний для составления базиса (В разработке для решётки)

Файлы plot - для генерации графиков (На стадии улучшения, на данный момент реализован matplotlib)

Файлы quantum_operators - функции эволюции заданного состояния под воздействием некоторых стандартных операторов. Подробнее в quantum_operators.hpp

# Параллельного программирования на MPI

Все примеры программы в examples, в которых используется MPI - с припиской _MPI.

В силу низкоуровневости самого MPI, использование автоматической параллелизации в данном ПО нетривиально.
Для работы автоматической MPI версии необходимо минимум 2 MPI процесса.

Внизу написан минимальный код для работы с 2, 3 версии библиотеки.

```
 #include "QComputations_MPI.hpp"
 #include <iostream>

 int main(int argc, char** argv) {
     using namespace QComputations;
 
 // ---------------- BIG MPI_Init -------------
     int rank, world_size;
     MPI_Init(&argc, &argv);
     MPI_Comm_size(MPI_COMM_WORLD, &world_size);
     //std::cout << "WORLD SIZE - " << world_size << std::endl;
     if (world_size == 1) {
         std::cerr << "Should have at least 2 processes\n";
 
         MPI_Finalize();
         return 1;
     }
 
     MPI_Comm_rank(MPI_COMM_WORLD, &rank);
 
     if (rank != 0) {
        // run_mpi_slaves() - запуск автоматического распараллеливания
        // !!! ВСЕ ПРОЦЕССЫ, КРОМЕ 1, ДОЛЖНЫ БЫТЬ В run_mpi_slaves() !!!
         mpi::run_mpi_slaves();
         MPI_Finalize();
         return 0;
     }
 // ------------------ end BIG MPI_Init --------


    std::cout << "Hello, world!" << std::endl;


    // Остановка остальных MPI процессов. (Просто выходят из run_mpi_slaves())
     mpi::stop_mpi_slaves();
     MPI_Finalize();
     return 0;
 }
```

После инициализации MPI, программирование осуществляется абсолютно также, как и в 1 версии библиотеки (на 1 ядре), кроме основного квантового уравнения. (Временно)

(ВАЖНО!!!) В основной программе должен остаться только процесс с рангом 0 + должно быть запущено не менее 2 процессов.
